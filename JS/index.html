Classes Classes Classes A class in JavaScript is a blueprint for creating objects with similar properties and methods. It provides a cleaner and more structured way to define object-oriented concepts like inheritance, encapsulation, and abstraction.
Data Structures Map Map is a built-in object in JavaScript that allows you to store key-value pairs. Unlike plain objects, keys in a Map can be of any data type, including objects, functions, and primitives. The difference between Map and Object is that Map is a collection of key-value pairs (string, number, boolean etc), while Object is a collection of properties (string and symbol).
Weak Map Weak Map is a specialized type of Map in JavaScript that holds weakly referenced key-value pairs
Set Set is a built-in JavaScript object that stores unique values of any type, whether primitive values or object references. Unlike arrays, Sets automatically eliminate duplicate values.
Weak Set WeakSet is a special type of Set in JavaScript that holds weak references to its elements. Unlike a regular Set, the values stored in a WeakSet are objects, and they do not prevent garbage collection when there are no other references to the object. This means that if an object in a WeakSet is no longer referenced elsewhere in your code, it can be garbage collected.
JSON JSON (JavaScript Object Notation) is a lightweight data interchange format that is easy for humans to read and write and easy for machines to parse and generate. It is widely used for transmitting data between a server and a client in web applications.
Error Handling try try block is used to enclose code that may potentially throw an error. It allows you to test a block of code for errors without crashing the program.
Catch Catch block follows the try block. It executes only if an error occurs in the try block. It allows you to handle the error, log it, or recover from it gracefully.
finally finally block is an optional block that runs after the try and catch blocks, regardless of whether an error occurred or not. It is commonly used to clean up resources, such as closing files or network connections. Even if no error is thrown, the finally block will still execute
thrown thrown statement is used to manually throw an error. You can throw custom errors or use it to throw built-in JavaScript errors, like Error, TypeError, etc.
Error Object Error Object in JavaScript represents runtime errors. It is used to create error objects with useful information about the error, like a message and an optional stack trace.
Asynchronous JavaScript Promises Promises is an object that represents the eventual completion (or failure) of an asynchronous operation and its resulting value. Promises provide a cleaner way to handle asynchronous code compared to callbacks. They can be in one of three states: pending, resolved (fulfilled), or rejected.
