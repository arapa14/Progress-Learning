Type Assertion as keyword Used to explicitly set the type of a value or variable
<> syntax Another way to perform a type assertion.
Interfaces defining an interface specifies the shape of an object, including the properties and their types
Extending interfaces can inherit properties from other interfaces using the extends keyword. its like combine an interfaces with another interfaces
optional properties optional properties are denoted with a question mark (?) after the property name, meaning they may or may not be present
readonly properties ensures that a property cannot be moodified after it is set
Dynamic keys allow you to define properties eith keys of certain type or pattern using index signatures
classes properties variable defined in class to hold data or state
methods function defined within a class that perform operations
constructor a special method called when a new instance of the class is created. It initializes properties.
Inheritance inheritance allows a class to extend another class and inherit its properties and methods using the extend keyword
access modifier (public) accessible everywhere (default if no modifier is specified).
access modifier (private) accessible within the class only
access modifier (protected) accessible within the class and its subclasses
function type annotations specify the types of parameter and the return value of function
optional parameters optional paramters are denotade by a question mark (?). These parameters are not required when calling the function
default parameters provide a default value if no value is passed during the function call
rest parameter allow a function to accept an arbitrary number of arguments as an array
Generic generic function A generic function is defined using a type parameter, usually denoted by (or any identifier). This type parameter acts as a placeholder for the actual type provided when the function is invoked.
generic classes Generic classes work similarly to generic functions. They allow a class to operate with a type parameter.
modules import to use exports from another module, you import
export you can export variables, function, classes, or interfaces from a module so that other files can use them
namespace Namespaces are an older way of organizing code in TypeScript. They group related functionality under a single name, helping avoid naming conflicts. Unlike modules, namespaces are primarily for internal organization and are not recommended for modern TypeScript projects.
utility types partial makes a property of a type optional
Pick Selects specific properties from a type.
Omit Excludes specific properties from a type.
Readonly Makes all properties of a type read-only.
Record Constructs a type with keys mapped to a specified type.
Exclude Removes specific types from a union.
Extract Selects specific types from a union.
NonNullable Removes null and undefined from a type.
Parameters Extracts the parameter types of a function as a tuple.
ReturnType Extracts the return type of a function.
InstanceType Extracts the type of an instance of a class.
Required Makes all properties of a type required.
ThisType Specifies the type for the this keyword in an object literal.
