GIT Fundamental
Section Task Description
Setup and configuration git init Initialize a new Git repository
git clone clone an existing git repository
git config configures git setting
Staging git status display current state of the working directory and staging area
git add Adds change to staging area, preparing them for a commit
git rm removes files from the working directory and the repository
git mv renames omoves a file and stages the change
git commit save change from staging area to the repository
git reset unstages or reverts changes in the repository
inspect and compare git log display the commit history of the repository
git diff shows differences between changes in the repository
git show display detailed information about a specific commit
Branching git branch manages branch in repository
git checkout switches between branches or restore files
git merge combine change from one branch into another
remote repository git remote Manages connections to remote repositories
git fetch Downloads the latest changes from a remote repository without merging them into the local branch.
git pull fetches and merges the latest change from a remote repository into the local branch
git push uploads local changes to the remote repository
temporary commits git stash Temporarily saves changes in your working directory that are not yet staged, allowing you to switch branches or work on something else without losing your progress. The stashed changes can be reapplied later.
Github fork a fork is a copy of a repository that is created under your own account, allowing you to make changes without affecting the original repository
pull request a request to merge changes from one branch or repository into another
code review a process where developers review each other's code to ensure quality, maintainability, and adherence to standarts
